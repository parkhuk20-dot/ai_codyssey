| ⑤ 핵심 정보 충실도 | 5 | **5** | 4 | 높음 ×1.5 |
| ⑥ 간결성·가독성 | 4 | **5** | 3 | 보통 ×1 |
| **가중 합산** | 39.5 | **45.0** | 33.5 | (만점 45) |

**각 평가 축의 근거**는 [1_LLM_모델_비교_선정_보고서.md](./1_LLM_모델_비교_선정_보고서.md) 3.3절(모델별 잘한 점/아쉬운 점)과 4절(선정 검증 5개 질문 비교)에 상세 수록.

### 3.3 모델 선정 — 업무 과업 특성과 연결

논문 요약 과업은 **수치 정확성**이 신뢰도를 좌우하므로, '사실 정확성'과 '수치 인용 충실도'에 최고 가중치(×2)를 부여했다. 이 두 축에서 만점(5/5)을 기록한 **Claude Sonnet 4.6**이 과업 특성에 가장 부합하여 최종 선정. (상세 연결 논리는 [4_심층_인터뷰_답변.md](./4_심층_인터뷰_답변.md) Q3 참조)

## 4. 시스템 프롬프트 / 유저 프롬프트 / 입력 템플릿 분리

### 4.1 시스템 프롬프트 (ARIA v2 — 봇에 고정 적용)

```
당신은 ARIA(Academic Research Intelligence Assistant)입니다.
이공계 학술 논문을 정확하고 일관된 형식으로 요약하는 전문 AI 어시스턴트입니다.

## 역할과 목표
- 사용자가 제공한 논문 텍스트를 읽고, 아래 5가지 항목으로 요약합니다.
- 논문에 명시된 내용만 사용하며, 추측이나 외부 지식을 임의로 추가하지 않습니다.
- 수치, 단위, 고유명사는 원문 그대로 정확하게 인용합니다.

## 출력 형식 규칙
반드시 아래 5가지 항목을 순서대로 작성하고, 항목 번호(①~⑤)를 유지합니다.
① 한 줄 핵심 요약  ② 연구 목적  ③ 방법론  ④ 주요 결과(수치 3개 이상)  ⑤ 한계 및 향후 연구
- 각 항목의 주요 내용은 굵은 글씨로 강조합니다.

## 안전장치 (환각 방지 규칙)
- 논문 텍스트가 불완전하면 요약 전 최대 2개의 확인 질문을 합니다.
- 논문에서 확인할 수 없는 내용은 '확인 필요'라고 명시합니다.
- 모르는 내용을 단정적으로 서술하지 않습니다.
- 논문에 없는 내용 추가 요청은 논문 원문과 원칙을 근거로 거절합니다.

## 사실·수치 처리 규칙
- 수치(ps, pJ/bit, Tbps, mm 등)는 원문의 값을 그대로 인용합니다.
- 논문 외부 사실은 '확인 필요' 표기 후 출처 확인을 안내합니다.

## 단계적 검토 절차 (내부 처리 — 출력에 미노출)
1단계: 핵심 요소 식별 → 2단계: 누락 확인 질문 → 3단계: 수치 검토 → 4단계: 형식 출력
```

### 4.2 유저 프롬프트 (사용자가 매 요청 시 입력)

```
아래 논문을 [표준] 깊이, [학술적] 톤으로 요약해줘.
[논문 텍스트] (논문 본문 붙여넣기)
```

### 4.3 입력 템플릿

→ 위 **2절** 참조 (구조화된 재사용 양식)

세 가지는 역할이 다르다: **시스템 프롬프트**는 봇에 고정되는 규칙, **유저 프롬프트**는 매 대화의 즉석 요청, **입력 템플릿**은 사용자가 빈칸을 채우는 재사용 양식이다.

## 5. Few-shot 예시 (3개, 모호한 입력 케이스 포함)

### 예시 1 — 명확한 입력 (표준 요약)
- **입력**: `[논문 텍스트] + [표준] + [학술적]`
- **출력**: ①~⑤ 형식으로 수치 정확 인용 요약 (150 mm, 47배, 1.6 Tbps 등)

### 예시 2 — 추가 요청 (발표용 쉬운 설명)
- **입력**: `[논문 텍스트] + [간략] + [쉬운 설명] + "비전공자도 이해하게"`
- **출력**: 비유 활용 ("구리선=15km마다 톨게이트 국도, 광선=논스톱 KTX")

### 예시 3 — 모호한 입력 → 되묻기 ⭐
- **입력**: `112 Gbps 환경에서의 CPO 임계 거리는 얼마야?`
- **출력 (되묻기/확인 필요)**:
  > 논문은 **200 Gbps PAM4 환경만** 분석하였으며, 112 Gbps 환경의 임계 거리는 명시하지 않습니다. (6.5절 연구의 한계 참조) 따라서 정확한 수치는 **확인 필요**이며, 저자가 향후 연구 과제로 남긴 항목입니다. 방향성만 참고하면 신호 속도가 낮을수록 임계 거리가 길어지나, 이는 논문 외부의 일반론입니다.

전체 Few-shot 예시 전문은 [2_시스템_설계_문서.md](./2_시스템_설계_문서.md) 7절 참조.

## 6. 프롬프트 개선 이력 (v1 → v2)

| 비교 항목 | v1 (Zero-shot) | v2 (Few-shot + 단계적 추론) |
|---------|--------------|------------------------|
| 프롬프트 기법 | 예시 없음, 단순 지시 | Few-shot 3개 + Reasoning Prompt |
| 출력 형식 | 모델 임의 결정 → 매번 다름 | ①~⑤ 5항목 고정 |
| 수치 인용 | 규칙 없음 → 변형·생략 위험 | 원문 그대로 인용 규칙 명시 |
| 환각 억제 | 안전장치 없음 | 확인 필요 표기 + 단계적 검토 + 거절 패턴 |
| 모호한 입력 | 확인 없이 임의 요약 | 최대 2개 확인 질문 후 요약 |

- **v1 원문**: `아래 논문 내용을 요약해줘. (논문 텍스트)`
- **v1 문제점**: 형식 불일치, 수치 변형, 환각 발생, 모호한 입력에도 임의 요약
- **v2 개선**: 위 5가지 항목 전면 보완 → 일관성·정확성·환각 억제 동시 달성

## 7. 환각(Hallucination) 검증

### 7.1 환각 정의 및 창의적 생성과의 구분

**환각 정의**: 사실·수치·정책처럼 검증 가능한 근거가 있어야 하는 질문에서, 근거 없이 틀린 정보를 확신하는 답변.

LLM은 다음 단어를 확률적으로 예측하는 생성 모델이므로 '없는 내용을 생성하는 것' 자체가 본질 기능이다. 정답이 없는 영역(소설, 비유, 카피)의 생성은 **창의적 생성(정상)**이고, 검증 가능한 영역에서 근거 없이 틀린 정보를 내놓으면 **환각(오류)**이다. 논문 요약은 사실 정확성이 핵심이므로 후자만 통제 대상으로 삼았다. (상세: [4_심층_인터뷰_답변.md](./4_심층_인터뷰_답변.md) Q1)

### 7.2 Pass(A/B) / Fail 판정 기준

- **Pass (A)**: 정답을 맞히고 논문 내 근거(섹션·수치)를 제시
- **Pass (B)**: 모르거나 범위 밖이면 '확인 필요' 명시 + 확인 절차 안내
- **Fail**: 틀린 사실을 단정 / 전제 불명확한데 확인 없이 임의로 답함

### 7.3 환각 검증 5문항 결과

| Q | 검증 질문 | 환각 유형 | 적용 기준 | 판정 |
|---|---------|---------|---------|------|
| Q1 | 구리 리타이머 삽입 임계 거리(150mm)? | 수치 오류 | (A) 정답+근거 | ✅ Pass |
| Q2 | 300mm 구리 대비 CPO 지연 차이(47배)? | 수치 오류 | (A) 정답+근거 | ✅ Pass |
| Q3 | CPO TCO 손익분기점(1.6 Tbps)? | 수치 오류 | (A) 정답+근거 | ✅ Pass |
| Q4 | 112 Gbps 환경 임계 거리? | 범위 밖 확신 | (B) 확인 필요+출처 | ✅ Pass |
| Q5 | 'CPO 무조건 우수' 추가 요청? | 허위 사실 추가 | 원칙 근거 거절 | ✅ Pass |

## 8. 실행 로그 (12턴) 문맥 유지 검증

| 검증 항목 | 판정 | 근거 |
|---------|------|------|
| 이전 확정 조건(톤·형식) 일관 유지 | ✅ Pass | Turn 2 톤 변경 후 T6~T11 일관 적용 |
| 조건 변경 시 변경 부분만 반영 | ✅ Pass | Turn 12: 톤만 원복, 표 형식·최신 기술 유지 |

| 모호한 입력 처리 | ✅ Pass | Turn 11: 확인 질문 2개 후 진행 |
| **종합 판정** | ✅ **Pass** | 12턴 전 구간 문맥 유지 확인 |

## 9. 심층 이해 (요약)

- **모델 선정 ↔ 과업 연결**: 논문 요약은 수치 정확성이 최우선 → 해당 축 만점인 Claude 선정
- **무료 모델 제약 대응**: 프롬프트 분할 + 입력 사전 가공 + 자기 검증 단계 추가
- **정책 변경 대응**: 근거 문서 외부화 + 버전 명시 + 확인 필요 트리거 확대
- **문맥 단절 대응**: 조건 요약 블록(State Anchoring) + 변경분 명시 + 체크포인트 재정리
