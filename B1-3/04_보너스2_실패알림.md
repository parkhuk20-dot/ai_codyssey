# [보너스 2] 실패 알림 및 재시도 전략 — RSS 뉴스 수집 오류 감지 시스템

## 1. 개요

RSS 뉴스 수집 워크플로우에 **Error Handler + Retry 모듈**을 추가하여, HTTP 요청 실패 시 자동으로 재시도하고 오류 발생 즉시 Discord로 알림을 보내도록 구현했다.

---

## 2. 발생 가능한 오류 유형

| 오류 유형 | 원인 | 발생 상황 |
|-----------|------|-----------|
| RSS 피드 응답 없음 | 서버 다운, 타임아웃 | HTTP 요청 실패 |
| 잘못된 URL | URL 형식 오류, 피드 주소 변경 | BundleValidationError |
| AI 요약 실패 | OpenAI 응답 없음, 토큰 초과 | Generate a response 실패 |
| 데이터 파싱 오류 | RSS 구조 변경, 인코딩 문제 | 빈 값 반환 |

---

## 3. 워크플로우 구조

```
[Trigger] RSS — Watch RSS feed items (새 기사 감지)
   │
   ▼
[HTTP] Make a request (RSS 피드 URL 요청)
   │
   ├─(성공)→ [OpenAI] Generate a response (기사 AI 요약)
   │              │
   │              ▼
   │         [If-else] AI 요약 성공 여부 확인
   │              ├─ 1st (성공) → Discord 기사 알림 (7번)
   │              └─ Else (실패) → Discord 실패 알림 (6번)
   │
   └─(실패)→ [Error Handler]
                  ├─ Retry 자동 재시도 (3회, 1분 간격) (13번)
                  └─ Discord 오류 알림 즉시 전송 (14번)
```

---

## 4. 모듈 구성

| 모듈 번호 | 앱 | 기능 | 역할 |
|-----------|-----|------|------|
| 1 | RSS | Watch RSS feed items | 새 기사 감지 (Trigger) |
| 2 | HTTP | Make a request (GET) | RSS 피드 URL 요청 |
| 4 | OpenAI | Generate a response | 기사 내용 AI 요약 |
| 5 | If-else | 조건 분기 | AI 요약 성공 여부 확인 |
| 6 | Discord | Send a Message | AI 요약 실패 시 알림 (Else 경로) |
| 7 | Discord | Send a Message | 기사 정상 수신 알림 (1st 경로) |
| 13 | Retry | Flow Control | HTTP 실패 시 3회 자동 재시도 |
| 14 | Discord | Send a Message | HTTP 오류 즉시 알림 |

---

## 5. Error Handler 설정

### Retry 모듈 설정 (13번)

| 항목 | 설정값 |
|------|--------|
| Retry automatically | Yes |
| Number of retries | 3 |
| Minutes between retries | 1 |

3회 재시도 후에도 실패하면 Incomplete Executions에 기록되며 담당자가 수동 확인 가능.

### Store incomplete executions 설정

Scenario settings → **Store incomplete executions → Yes** 로 활성화하여 실패한 실행 데이터를 보존.
