# [보너스 2] 실패 알림 및 대체 경로 전략 — 연락처 형식 오류 감지 시스템

## 1. 개요

공연 예매 시스템 워크플로우에 **If-else 모듈**을 추가하여, 연락처 형식이 올바르지 않은 예매 신청을 자동으로 감지하고 오류 시트에 별도 기록한 뒤 담당자에게 Discord로 즉시 알림을 보내도록 구현했다.

---

## 2. 문제 정의

예매 신청 폼에서 연락처를 자유 입력으로 받다 보면 아래와 같은 형식 오류가 발생할 수 있다.

| 오류 유형 | 예시 |
|-----------|------|
| 하이픈 없이 입력 | 01012341234 |
| 잘못된 자릿수 | 010-1234-123 |
| 문자 포함 | 010-abcd-1234 |

이러한 오류 데이터가 정상 시트에 그대로 기록되면 나중에 담당자가 연락할 때 문제가 생긴다. 이를 자동으로 감지해 **정상 경로와 오류 경로를 분리**하고 즉시 알림을 보내는 전략을 설계했다.

---

## 3. 적용 시나리오

**공연 예매 시스템 (Make)**

```
[Trigger] Google Sheets — Watch New Rows
   │
   ▼
[If-else] 연락처 형식 검증
   │
   ├─ 정상 통과 (형식 올바름)
   │      └─ [Action] Google Sheets 정상 시트 기록
   │
   └─ Else (형식 오류)
          ├─ [Action 1] Google Sheets 오류 탭 기록 (대체 경로)
          └─ [Action 2] Discord 오류 알림 전송
```

---

## 4. Make 모듈 설정

### 사용 모듈
- **If-else** (Flow Control)
- 위치: Google Sheets Watch New Rows 바로 다음

### If 조건 설정 (정상 통과 조건)

| 항목 | 설정값 |
|------|--------|
| 조건 대상 | 연락처 열 |
| 연산자 | `Matches pattern` |
| 패턴 | `010-\d{4}-\d{4}` |

- 연락처가 `010-0000-0000` 형식이면 **정상 통과(1st)**
- 그 외 모든 형식은 **Else 경로**로 분기

---

## 5. 시나리오 전체 화면

If-else 모듈을 기준으로 정상 통과(1st)와 Else 두 경로가 분기되어 있다. 정상 통과는 기존 예매 처리 시트에 기록되고, Else는 오류 시트 기록 + Discord 알림으로 연결된다.

## 6. 실행 결과

### 테스트 케이스

연락처를 `01012341234` (하이픈 없음) 형식으로 입력한 신청을 제출하여 Else 경로가 정상 동작하는지 확인했다.

### Discord 오류 알림 수신

Else 경로로 분기된 데이터에 대해 Discord에 즉시 알림이 수신되었다.

- 오류 발생!
- 이름: 김지윤
- 학년: 3학년
- 전화번호: 01012341234

![Discord 오류 알림 수신 화면](bonus2_discord_alert.png)

### Google Sheets 오류 탭 기록

오류가 감지된 데이터는 정상 시트가 아닌 별도 오류 탭에 자동으로 기록되었다.

![오류 탭 기록 화면](bonus2_sheets_error.png)

---

## 7. 대체 경로 설계 의의

| 항목 | 처리 방식 |
|------|-----------|
| 정상 데이터 | 기존 예매 유형별 시트에 자동 기록 |
| 오류 데이터 | 별도 오류 탭에 격리 기록 + 담당자 Discord 알림 |
| 알림 방식 | Discord 즉시 전송 (실시간 감지) |
| 재처리 방법 | 담당자가 오류 탭 확인 후 수동 수정 및 재처리 |
