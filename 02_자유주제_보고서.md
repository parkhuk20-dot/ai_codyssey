# [프로젝트 2] 자유 주제 자동화 — 공연 예매 유형별 자동 분류 및 알림 시스템

## 1. 자동화할 반복 업무 정의

공연 예매 신청이 들어올 때마다 담당자가 ① 신청 내용을 확인하고 ② 예매 유형(일반/얼리버드/동문/초대권)을 분류해 ③ 해당 담당자에게 알리고 ④ 유형별 시트에 기록하는 작업을 매번 수작업으로 처리하고 있었다. 신청 건수가 많아질수록 누락·오분류 위험이 높아지고, 담당자의 반복 업무 부담도 커진다. **이 분류·알림·기록 과정을 자동화**하여 신청이 들어오는 즉시 올바른 채널로 알림이 가고 유형별 시트에 자동 기록되도록 한다.

---

## 2. 선정 도구와 이유 — Make

- 무료 플랜에서 **4-갈래 조건 분기(Router) + 멀티 Action**을 모두 구현 가능 (Zapier 무료는 분기 불가)
- 시각적 캔버스로 4개 분기 흐름을 한눈에 파악하고 관리하기 편리
- Google Sheets Watch New Rows 모듈로 폼 응답을 안정적으로 감지 가능
- 1,000 크레딧/월 무료 범위 내에서 충분히 운영 가능

---

## 3. 워크플로우 흐름

```
[Trigger] Google Sheets — Watch New Rows (폼 응답 시트 감지)
   │
   ▼
[Router] 예매 유형 분기
   ├─ 일반 예매      → [Action 1] Discord #일반예매 알림
   │                 → [Action 2] Google Sheets 일반예매 탭 기록
   │
   ├─ 얼리버드 예매  → [Action 1] Discord #얼리버드 알림
   │                 → [Action 2] Google Sheets 얼리버드 탭 기록
   │
   ├─ 건대극장 회원  → [Action 1] Discord #회원동문 알림
   │  &동문 할인     → [Action 2] Google Sheets 회원&동문 탭 기록
   │
   └─ 공연팀 초대권  → [Action 1] Discord #초대권 알림
                     → [Action 2] Google Sheets 초대권 탭 기록
```

- **Trigger 자동 실행**: 시나리오를 ON 상태로 두어 폼 제출 시 자동 기동
- **분기 4경로 실행 확인**: 각 유형으로 1건씩 제출해 4개 경로 모두 실행 로그 확보

## 4. Google Form 구성

| 질문 | 형식 | 비고 |
|------|------|------|
| 성함 | 장문형 | 필수 |
| 연락처 | 장문형 | 필수, 예) 010-1234-1234 |
| 관람 회차 | 객관식 | 1~5회차 선택 |
| 예매 유형 | 객관식 | 일반 예매 / 얼리버드 예매 / 건대극장 회원&동문 할인 예매 / 공연팀 초대권 |
| 예매 수량 | 단답형 | 숫자, 필수 |

## 5. Make 시나리오 구성

### 모듈 구성

| 모듈 번호 | 앱 | 기능 | 설정 |
|-----------|-----|------|------|
| 1 | Google Sheets | Watch New Rows | 폼 응답 시트 감지 |
| 2 | Router | 분기 | 예매 유형 기준 4갈래 |
| 3 | Discord | Send a Message | 일반예매 채널 알림 |
| 4 | Google Sheets | Add a Row | 일반예매 탭 기록 |
| 5 | Discord | Send a Message | 얼리버드 채널 알림 |
| 6 | Google Sheets | Add a Row | 얼리버드 탭 기록 |
| 7 | Discord | Send a Message | 회원&동문 채널 알림 |
| 8 | Google Sheets | Add a Row | 회원&동문 탭 기록 |
| 9 | Discord | Send a Message | 초대권 채널 알림 |
| 10 | Google Sheets | Add a Row | 초대권 탭 기록 |

### Router 필터 조건

| 경로 | 조건 |
|------|------|
| 1st | 예매 유형 = 일반 예매 |
| 2nd | 예매 유형 = 얼리버드 예매 |
| 3rd | 예매 유형 = 건대극장 회원&동문 할인 예매 |
| 4th | 예매 유형 = 공연팀 초대권 |

## 7. 자동화 효과

| 항목 | 자동화 전 | 자동화 후 |
|------|-----------|-----------|
| 분류 방식 | 담당자가 수동으로 유형 확인 | 폼 응답 즉시 자동 분류 |
| 알림 방식 | 담당자가 직접 채널에 공유 | Discord 해당 채널 자동 전송 |
| 기록 방식 | 수동으로 시트에 입력 | 유형별 탭에 자동 기록 |
| 오류 가능성 | 누락·오분류 위험 있음 | 조건 기반 자동 처리로 오류 최소화 |
